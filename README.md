# Job Search

## Описание
Проект представляет собой инструмент для сбора, хранения и анализа данных о компаниях и вакансиях
с сайта `hh.ru`. В рамках работы реализовано взаимодействие с публичным `API` `hh.ru`, проектирование 
и автоматическое создание базы данных `PostgreSQL`, а также загрузка полученной информации в 
структурированные таблицы.

## Функционал
Приложение предоставляет пользователю интерактивное меню для доступа к данным. 
Ниже подробно описаны доступные операции.

### 1. Показать компании и количество их вакансий.
Эта функция выводит список компаний (из предопределенного набора: `Sber`, `Ozon Tech`, `Yandex`, `VK`, `Tinkoff`)
и количество вакансий, открытых в каждой из них.

### 2. Вывести список всех вакансий. 
Выводит полный список вакансий, хранящихся в базе данных.

Что отображается: Для каждой вакансии выводится её название, название компании-работодателя, 
вилка зарплаты (от/до), валюта и ссылка на вакансию на `hh.ru`.

### 3. Узнать среднюю зарплату
Рассчитывает среднюю заработную плату по всем вакансиям в базе.

### 4. Найти вакансии с зарплатой выше средней
Фильтрует вакансии, предлагая только те, где средняя зарплата 
(рассчитанная как `(salary_from + salary_to) / 2)` превышает общий средний показатель по базе.

### 5. Поиск вакансий по ключевому слову
Ищет вакансии по названию, используя оператор `LIKE`.

## Установка
Для установки проекта выполните следующие шаги:

### Клонируйте репозиторий:
```
git clone https://github.com/kiravolkova7-code/job_search.git
```
### Создайте виртуальное окружение (рекомендуется):

```
python -m venv venv
```
* Для Windows:
``` 
.\venv\Scripts\activate 
```
### Установите зависимости:

```
pip install -r requirements.txt
```
### Настройте подключение к БД:
Создайте файл `.env` в корне проекта на основе `.env_example`.
Укажите параметры подключения к вашей базе данных `PostgreSQL`:

```
DB_HOST=localhost
DB_PORT=5432
DB_NAME=hh_vacancies_db
DB_USER=your_user
DB_PASSWORD=your_password
HH_API_USER_AGENT=Mozilla/5.0 (compatible; MyVacancyParser/1.0)
```

## Зависимости
Для работы проекта требуются следующие библиотеки Python:

`psycopg2` — драйвер для работы с `PostgreSQL`.

`requests` — библиотека для выполнения `HTTP`-запросов к `API` `hh.ru`.

`python-dotenv` — для загрузки переменных окружения из файла `.env`.

`pytest` — для запуска тестов.

Список зависимостей находится в файле `requirements.txt`.

## Тестирование
Проект покрыт модульными тестами с использованием фреймворка `pytest`. 
Тесты гарантируют корректность работы ключевых компонентов приложения: взаимодействия с БД, парсинга `API`
и работы с файлами `JSON`.

Фикстуры:
* `db_manager`: Создает временную тестовую базу данных перед запуском тестов, 
инициализирует таблицы и удаляет БД после завершения всех тестов.
* `clean_db`: Автоматически очищает таблицы перед каждым тестом. 
Это гарантирует независимость тестов друг от друга (отсутствие побочных эффектов).

Мокирование:
* Для изоляции тестов от внешних сервисов (`API` `hh.ru`) используются моки (`unittest.mock`). 
Методы классов `HHApiWorker` и `JSONWorker` подменяются на заглушки, возвращающие предопределенные данные.

Проверка работы с БД:
* Тесты проверяют успешное создание таблиц (`test_create_tables`).
* Тестируется вставка новых записей в таблицы работодателей и вакансий 
(`test_insert_employer`, `test_insert_vacancy`).
* Проверяется корректность `SQL`-запросов на выборку данных: получение всех вакансий, 
средней зарплаты и фильтрация по зарплате.

Проверка обработки ошибок:
* Тесты проверяют поведение приложения при сетевых сбоях (например, при недоступности `API` `hh.ru`). 
Ожидается, что методы вернут пустой список или корректно обработают исключение.

Тестирование вспомогательных функций:
* Проверяется сборка конфигурации БД из переменных окружения (`test_get_db_config`).
* Тестируется функция проверки доступности сервиса PostgreSQL (`test_check_postgres_service`).

## Запуск тестов
Для запуска всех тестов выполните команду в корневой директории проекта:

```
pytest
```
Тесты автоматически очищают тестовую базу данных перед каждым запуском, чтобы обеспечить их независимость.

## Использование
После установки зависимостей и настройки `.env` файла, вы можете запустить основной скрипт приложения. 
Точкой входа является функция `user_interface`, которая предоставляет консольное меню для взаимодействия с данными.

Запуск:
```
python src.main
```
